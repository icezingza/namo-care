4. System Flow — การทำงานทีละขั้น
4.1 Main Message Processing Flow
ทุกข้อความที่ผู้ใช้ส่งผ่าน LINE จะผ่านขั้นตอนนี้เสมอ (< 3 วินาที target)
1
รับ Webhook Event จาก LINE
LINE ส่ง POST ไปยัง Firebase Cloud Function endpoint ทันทีที่ผู้ใช้ส่งข้อความ ระบบ verify LINE signature ก่อนทุกครั้ง จากนั้น respond 200 OK ทันที (LINE timeout = 30s)
2
Emergency Detection (First Check)
ตรวจสอบ keyword ฉุกเฉินทั้งไทยและอังกฤษ ถ้า match → ส่งข้อความ crisis ให้ผู้ใช้ทันที + push alert ครอบครัวทุกคน + log alert record → END (bypass AI)
3
Load Context (Parallel)
โหลด Session State, User Memory (L1+L2+L3 compressed), Risk Score ปัจจุบัน ทำแบบ parallel Promise.all() เพื่อลด latency
4
AI Analysis
ส่งข้อความ + context ไปยัง Namo Nexus API วิเคราะห์: Emotion (label + confidence score), Intent (ประเภทของข้อความ), Keywords สำคัญ
5
Companion Mode Routing
เลือก mode ที่เหมาะสม: Emotional (รับฟัง) / Health (ยา/routine) / Cognitive (กิจกรรมสมอง) ตาม time-of-day + risk state + intent
6
Response Generation
Build system prompt จาก memory + persona + risk level → call Namo Nexus → validate response (safety check 5 layers) → simplify ถ้ายาวเกิน
7
Reply to User
ส่ง reply message ผ่าน LINE replyToken (หรือ pushMessage ถ้า reply expired) พร้อม rich menu / quick reply ถ้าเหมาะสม
8
Persist + Risk Update (Async)
บันทึก conversation log, update lastActiveAt, update session state, อัปเดต memory (ถ้าพบ significant event), คำนวณ risk score ใหม่ — ทำ async ไม่บล็อก response
